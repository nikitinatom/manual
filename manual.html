Приложение 1.

Сценарий сбора информации с веб-страницы.

1.Страница «Авторизация»

Для авторизации в системе требуется ввести имя пользователя и пароль и нажать кнопку «SIGN IN».

Рис. 1 – Страница «Авторизация»

  1. Ввести учётные данные.
  2. Нажать кнопку «SIGN IN».
  3. Откроется страница «Аналитика».

2. Страница «Аналитика»

Отображает статистику текущего проекта по темам, рубрикам и источникам. В части «Важное» отображаются темы с настроенной важностью. Тональность рассчитывает отношение проанализированных карточек новостей. Нажатие на элементы из частей «Важное», «Тональность сообщений» и «Динамика сообщений» применяет фильтр к ленте и открывает новое окно браузера.

Рис. 2 – Страница «Аналитика»

  1. Нажать на узел графика.
  2. Выполнен переход на страницу «Лента новостей» с фильтрацией по выбранному параметру.

Рис. 3 – Страница «Аналитика». Динамика сообщений

  1. Нажать на переключатель отображения графика.
  2. Нажать на название рубрики или темы для фильтрации.
  3. Нажать на точку графика для перехода в ленту с фильтрацией новостей по дате и выбранной теме или рубрике.

Рис. 4 – Страница «Аналитика». Важное и Тональность сообщений

  1. Навести курсор на элемент из части «Важное» для отображения описания новости.
  2. Нажать на элемент из части «Важное» для перехода в ленту с фильтрацией по теме.
  3. Навести на элемент из части «Тональность сообщений» для перехода в ленту с фильтрацией по тональности.

Рис. 5 – Страница «Аналитика». Кнопка выгрузки отчёта

  1. Нажать кнопку «Выгрузить отчёт» для автоматического скачивания отчёта.

3. Страница «Лента»

Рис. 6 – Страница «Лента». Общий вид

Лента графически делится на три столбца. В левом отображаются созданные в проекте рубрики и темы, по центру отображаются карточки в компактном виде. В правой части открывается подробная информация о выбранной карточке.

Рис. 7 – Страница «Лента». Полное отображение и развёрнутый текст карточки

  1. Нажать на карточку в центральном столбце для её полного отображения.
  2. Нажать «Читать полностью...» под текстом карточки в правом столбце для отображения текста целиком.

Над средним столбцом находятся кнопки управления: выделение всех отображаемых карточек, создание отчёта по выбранным карточкам, удаление карточки из системы, счётчик записей и страниц, кнопка меняющая упорядочивания карточек от старых к новым и наоборот.

Рис. 8 – Страница «Лента». Выбранные карточки. Счётчик и переключатель страниц

  1. Нажать на изображение аватарки автора для выбора карточки.
  2. Нажать стрелочки рядом со счётчиком карточек и страниц для перехода по страницам.
  3. Нажать «Упорядочить по возрастанию» для отображения ленты от старых карточек к новым.

Рис. 9 – Страница «Лента». Кнопки создания отчёта, удаления и выделения карточек

  1. Отметить карточки выбранными.
  2. Нажать кнопку выгрузки отчёта.
  3. Подтвердить создание отчёта.
  4. Система оповестит о начале создания отчёта.
  5. Загрузка отчёта начнётся автоматически.

На развёрнутом виде карточки цветной полосой отображается цвет важности, под заголовком указана дата публикации (дата публикации, указаная изданием, отобразится, если источник настроен на её сбор), расположены кнопки редактирования, добавления в «Избранное», удаления и выключения подсветки ключевых слов. В нижней части карточки система отобразит найденные данные, такие как Телефон, Хэш-теги, ссылки и другие.

Рис. 10 – Страница «Лента». Развёрнутый вид карточки

Для создания и изменения карточки используется редактор. В редакторе указываются темы, важность, тональность, дата и время публикации. Ввод ссылки на изображение прикрепит его к тексту карточки. Во вложение кнопками «Выберите файл» и «ЗАГРУЗИТЬ» добавляются файлы с компьютера.

Рис. 11 – Страница «Лента». Редактор карточек

Для поиска по ленте используется «Фильтр». Параметры фильтрации используют свойства карточек, источники, системные теги и время.

Рис. 12 – Страница «Лента». Фильтр

  1. Нажать«Фильтр» для вызова окна выбора условий фильтрации.
  2. Выбрать необходимые параметры.
  3. Нажать «ПРИМЕНИТЬ» для отображения ленты новостей с применением выбранных условий фильтрации.
  4. Нажать «Фильтр».
  5. Нажать «СБРОСИТЬ»: лента новостей отобразится по умолчанию.

Нажатие на названия рубрик и или тем также применяет фильтр к ленте. Рядом с элементами списков находятся кнопки «Выбрать все» или «Сбросить», если выбрано.

Рис. 13 – Страница «Лента новостей», фильтрация по рубрикам и темам.

4. Экран руководителя

Представляет собой лаконичное и актуальное отображение кратких карточек по темам с цветовой индикацией по важности. Экран обновляется автоматически.

Рис. 14 – Раздел «Лавина пульс». Общий вид

Для настройки экрана нажать «Редактировать блоки». Каждый блок имеет название, место расположения на экране, свойства видимости и автопубликации.

Рис. 15 – Раздел «Лавина пульс». Редактирование блоков

Автопубликация устанавливается для автоматического отображения карточек, прошедших рубрикацию. Рубрики, карточки которых требуется отображать выбираются из списка.

Рис. 16 – Раздел «Лавина пульс». Добавление рубрик для автопубликации

  1. Нажать «Редактировать блоки»: блоки поменяют внешний вид, станет доступна их настройка.
  2. Нажать кнопку «+»: появится новый пустой блок.
  3. Заполнить название и настройки блока.
  4. Выбрать положение блока на экране стрелками внизу блока.
  5. Нажать «УДАЛИТЬ БЛОК» в существующем блоке: блок будет удалён.
  6. Нажать «СОХРАНИТЬ»: система отобразит уведомление.
  7. Нажать «НАЗАД»: блоки поменяют внешний вид на обычное представление.

Для добавления карточки используется редактор.

Рис. 17 – Страница «Лавина пульс». Добавление новостной карточки

  1. Нажать «Добавить карточку»: откроется окно добавления карточки.
  2. Заполнить поля формы, нажать «СОХРАНИТЬ».

Созданная карточка появится в ленте новостей и в разделе «Лавина пульс».

5. Страница «Рубрики»

В рубриках указываются поисковые запросы, по которым система будет проводить отбор новостей.

Рис. 18 – Страница «Рубрики»

Для создания рубрики используется одноименная кнопка «Создать рубрику», после её нажатия система предложит ввести название новой рубрики.

Рис. 19 – Страница «Рубрики». Окно создания рубрики

  1. Нажать «Создать рубрику»: отобразится окно создания рубрики.
  2. Ввести название, нажать «Сохранить».
  3. Новая рубрика появилась в списке слева.

Настройка рубрик включает в себя запрос, выбор важности, тональности и популярности, может применяться для избранных источников и уведомлять о карточках, которые прошли рубрикацию.

Рис. 20 – Страница «Рубрики». Окно настроек рубрики

6. Страница «Задачи»

Раздел «Задачи» служит для создания графов связей пользователей или групп и поиска профилей в социальных сетях. В кратком отображении списка задач отображаются название, тип, статус и данные о выполнении.

Рис. 21 – Страница «Задачи». Общий вид

Рис. 22 – Страница «Задачи», добавление новой задачи

После заполнения необходимых полей задача появится в списке.

После успешного выполнения задачи станет доступен результат выполнения:

Рис. 23 – Страница «Задачи». Результат выполнения задачи

Кнопка «ДЕТАЛИ» откроет граф в новом окне браузера.

Рис. 24 – Страница «Задачи». Граф выполненной задачи

Граф масштабируется колёсиком мыши, перетаскиванием узлы располагаются вручную, длина связей регулируется нажатием кнопок «+» и «-», справа расположены фильтры и кнопка создания отчёта. Нажатие на узел развернёт информацию о нём слева. Имя узла является ссылкой на профиль социальной сети.

  1. Выбрать завершенную задачу из списка, кликнуть на неё. Справа отобразится информация о задаче. Если задача выполнялась несколько раз, выбрать интересующие результаты из списка.
  2. Нажать «ДЕТАЛИ»: задача с построением графов откроется в новом окне браузера.
  3. Выбрать условия фильтрации справа. Неподходящие условиям фильтров узлы на графе станут прозрачными.
  4. Нажать на узел графа. Слева отобразится информация об узле. Имя выбранного узла является ссылкой на профиль в социальной сети.
  5. Нажать «ОТЧЁТ». Предварительный просмотр отчёта откроется в новом окне.

Рис. 25 – Страница «Задачи». Информация об узле

В отчёте приведены графики и другая информация.

Рис. 26 – Страница «Задачи». Элементы отчета

  1. Нажать кнопку добавления задачи: откроется окно создания задач.
  2. Из выпадающего списка выбрать тип задачи. Окно изменится, предлагая заполнить необходимые условия.
  3. Заполнить поля окна создания задачи, нажать «СОХРАНИТЬ».
  4. Задача появится в списке.

7. Страница «Досье»

Рис. 27 – Страница «Досье». Общий вид

В досье вводится информация о Персонах, Компаниях или Событиях. Запрос, созданный для объекта, отображает новости.

Рис. 28 – Страница «Досье». Создание нового объекта

  1. Нажать кнопку добавления объекта: отобразится окно выбора типа объекта.
  2. Выбрать необходимый тип объекта, нажать кнопку «Далее».
  3. Заполнить поля, предлагаемые для выбранного типа объекта, нажать «Далее».
  4. Проверить правильность заполнения, нажать «ГОТОВО». объект появится в списке.

Создав поисковый запрос на вкладке «Общее», во вкладке «Новости» будут отображаться карточки проекта, в которых присутствуют слова запроса.

Рис. 29 – Страница «Досье», добавление поискового запроса

  1. Выбрать объект, нажать «Редактировать».
  2. Указать поисковый запрос.
  3. Нажать «Сохранить».
  4. Перейти во вкладку «Новости»: отобразится список карточек, содержащих текст запроса.
  5. Нажать на карточку из списка: отобразится полное представление карточки.

Рис. 30 – Страница «Досье», отображение карточек по поисковому запросу

Во вкладке «Связи» добавляются связи с другими объектами раздела «Досье», «Гео» добавит место на карте, возможно добавление заметок и документов с компьютера пользователя.

В схему связей можно добавлять только объекты из «Досье», указывая тип связи. После сохранения схема связей будет доступна по кнопке «Схема связей».

  1. Перейти во вкладку «Связи».
  2. Нажать «Добавить связь».
  3. Выбрать тип связи и объект.
  4. Нажать «Сохранить».
  5. Нажать «Схема связей» для просмотра.

8. Страница «Источники»

Рис. 37 – Страница «Источники»

Источники проекта служат для явного указания места сбора информации. Для создания источника нужно нажать кнопку, указать тип источника и заполнить поля.

Рис. 38 – Страница «Источники». Добавление источника

Источник появится в списке.

Рис. 39 – Страница «Источники». Список источников

Источник включается и выключается кнопкой справа, добавление или редактирование тегов доступно при нажатии на иконку ярлыка, переход по ссылке источника – кнопкой рядом с названием источника.

Рис. 40 – Страница «Источники». Редактор тегов

По клику на имя источника открывается раздел его настройки, где реализованы настройки расписания, атрибуты, редактирование тегов и производится тестирование источника.

Рис. 41 – Страница «Источники». Настройки источника

В зависимости от выбранного типа окно создания отображает различные поля для заполнения:

  1. Заполнить поле «НАЗВАНИЕ» и остальные условия.
  2. Нажать «СОХРАНИТЬ», после чего источник появится в списке.
  3. Нажать на название источника. Открылось окно настройки источника.
  4. Во вкладке «Настройка источника» нажать «включить». Поле поменялось на зелёный цвет и источник включился. Управление, также, доступно из списка источников.
  5. Нажать «СОХРАНИТЬ ИЗМЕНЕНИЯ». Карта действий сохранилась.
  6. Во вкладке «Тестирование источника» нажать «RUN:GRAB»для запуска тестирования без использования JavaScript или «RUN:PHANTOMJS», если для сбора используются шаги, требующие выполнения скриптов.

Рис. 42 – Страница «Источники». Тестирование источника

9. Страница «Сканер»

Рис. 43 – Страница «Сканер»

На экране окно «Add task»

  1. Необходимо ввести URL ресурса, выбрать тип проверки, установить флажок «на контроль», если требуется периодическая проверка.
  2. Нажать «ДОБАВИТЬ», задача появится в списке.
  3. Нажать на задачу из списка для отображения подробностей справа.
  4. Нажать «Выгрузить отчёт», скачивание отчёта начнётся автоматически.

Рис. 44 – Страница «Сканер». Добавление задачи

Существует три типа проверок. При постановке «на контроль» задача будет периодически выполняться заново (период выполнения «раз в неделю»).

Кнопка «Выгрузить отчёт» начнёт загрузку отчёта по выполненной задаче, а кнопка «Удалить» – удаляет.

Рис. 45 – Страница «Сканер». Результаты выполнения задачи

Страница «Отчеты»

Рис. 47 – Страница «Отчеты»Общий вид

11. Операторы запросов рубрик

В кавычках указывается запрос, который должен присутствовать в тексте буквально, знаком тильда (~) указывается расстояние между словами, для поиска слов в обратном порядке минимальное число должно быть равно двум.

Рис. 53 – Страница «Рубрики». Оператор кавычки

Для поиска слов с опечатками или с окончаниями знак тильда ставится после слова.

12. Дополнительные правила для рубрик

Правилом может служить URL или тег (для нескольких URL). Если карточка подходит под правила, будет совершено выбранное действие:

  • «Только карточки проекта» — установив этот чекбокс, рубрикацию будут проходить только карточки из источников проекта, источники базы будут игнорироваться.
  • «Только указанные правила» — настройка служит для игнорирования не попадающих под указанные правила карточек или источников.

Карточки отображают цвет «Важности» и «Тональность», если эти параметры указаны вручную, прошли рубрикацию по правилам или в автоматическом режиме.

Приложение 1

Сценарий сбора информации с веб-страницы представлен в виде фрагментов программного кода. Весь сценарий и блоки с действиями заключается в скобки […], действия робота (шаг) -- в фигурные {…}. Для указания роботу места расположения информации используется язык запросов к элементам XML-документа XPath (XML Path Language).

Описание шагов сценария

  1. Шаг сценария «Ввод значения»

{

  "ignore_errors": false,        //игнорировать ошибки (true/false)

  "sleep": 0,               //задержка перед шагом (сек.)

  "type": "input",                  //тип шага сценария «Ввод значения»

  "value": "",             //значение для ввода

  "xpath": ""              //XPath поля для ввода данных

}

 

  1. Шаг сценария «Клик по элементу»

{

  "ignore_errors": false,        //игнорировать ошибки (true/false)

  "sleep": 0,               //задержка перед шагом (сек.)

  "type": "click",                   //тип шага сценария «Клик по элементу»

  "xpath": ""              //XPath элемента

}

 

  1. Шаг сценария «Блок ссылок»

{

  "ignore_errors": false,        //игнорировать ошибки (true/false)

  "limit": 10,              //ограничение на сбор элементов

  "sleep": 0,               //задержка перед шагом (сек.)

  "takers": [],             //блок тейкеров(если тейкеров не выбрано, блок остается пустым)

  "type": "transfer",   //тип шага сценария «Блок ссылок»

  "xpath": ""              //абсолютный XPath ссылок для сбора

}

 

  1. Шаг сценария «Блок элементов»

{

  "ignore_errors": false,        //игнорировать ошибки (true/false)

  "limit": 10,              //ограничение на сбор элементов

  "sleep": 0,               //задержка перед шагом (сек.)

  "takers": [],             //блок тейкеров(если тейкеров не выбрано, блок остается пустым)

  "type": "list",                      //тип шага сценария «Блок элементов»

  "xpath": ""              //XPath элементов, между которыми осуществляется сбор данных

}

  1. Шаг сценария «Захват данных»

{

  "ignore_errors": false,        //игнорировать ошибки (true/false)
